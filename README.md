# AirMock Mobile Pretty v9

v8 흐름을 기반으로 마지막 보딩패스 발급 화면을 이메일 발송 방식에서 PNG 저장 방식으로 변경한 버전입니다. `index.html`을 브라우저에서 열면 됩니다.

## 반영 사항

- v8의 탑승객 정보 입력 → 좌석 선택 → 기내식 선택 → 결제 확인 → 보딩패스 흐름 유지
- 이메일 입력란, 이메일 미리보기, 메일 발송 UX 제거
- 마지막 화면에 `전체 PNG 저장` 버튼 추가
- 각 보딩패스 아래에 `이 보딩패스 PNG 저장` 버튼 추가
- 전체 저장 시 발급된 보딩패스 수만큼 PNG 파일을 각각 순차 다운로드
- PDF 생성 기능은 넣지 않고 PNG 저장만 제공
- 선택 기내식이 있는 경우 보딩패스에 `기내식-메뉴명` 표시 유지
- DEMO 안내 유지: 실제 항공권/결제/탑승권이 아님

## 파일 구조

```text
index.html
css/style.css
js/app.js
data/seed-data.js
data/meal_table.json
assets/meal/*.png
```

## 참고

PNG 저장은 `html2canvas` CDN을 사용합니다. Vercel 배포 주소에서는 그대로 작동합니다. 로컬에서 테스트할 때도 인터넷 연결이 필요합니다.
여러 장을 한 번에 저장할 때 브라우저가 여러 파일 다운로드 허용을 요청할 수 있습니다.
